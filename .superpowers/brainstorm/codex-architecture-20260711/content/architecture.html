Kiến trúc đề xuất cho Daily Meal Flutter

Contract-first + vertical slices, dùng chung codebase cho Android và Web

Luồng tổng thể
Flutter Presentation
Material 3, responsive mobile/tablet/web
Feature Modules
Auth · Feed · Create/AI · Social · Chat · Premium · Admin
Application State
Riverpod AsyncNotifier + immutable states
⇄
Data Layer
Dio client · repositories · DTO/domain mapping
Realtime & Device
Socket.IO · secure storage · media · notifications
Production Backend
https://api.dailymeal.site · REST + WebSocket

Thứ tự vertical slices

1. Core/API contract→ 2. Auth & onboarding→ 3. Feed & post→ 4. Create & AI→ 5. Social→ 6. Chat & notifications→ 7. Premium→ 8. Admin

Nguyên tắc bắt buộc

  • Không mock API khi endpoint thật đã tồn tại
  • Mỗi slice phải chạy end-to-end trước khi chuyển tiếp
  • Tái sử dụng logo, ảnh, icon và nội dung từ app gốc
  • Android và Web đều có loading/error/empty/offline states

Không thực hiện

  • Không sửa hoặc xây lại Node.js backend nếu contract vẫn đáp ứng
  • Không dịch máy từng component React Native sang Dart
  • Không để placeholder, TODO hoặc nút không hoạt động
  • Không bỏ Admin vì ưu tiên mobile
